# 角色与关注面（合成样例，用于干预评测）

> 评测时可将本节追加到 `runtime/context/instance-calibration-notes.md`（或与邮箱无关时写入 `manual-facts.yaml` 的 role_hint 类事实）后重跑 Phase 4。

## 我的角色

- 部门：交付部  
- 岗位：技术支持 / 运维开发工程师  

## 我通常**不负责**的事

- 资源申请审批链上的「拍板」；不是资源审批的第一责任人。

## 我**持续关注**的事

- **个人工时**：填报截止、填满率提醒、补填与审批节奏。  
- **集团公共事件通知**：公司级/集团级群发、安全与合规通告。  
- **HR / 人事相关**：假期、证明、流程类邮件。  
- **授权与权限**：账号、证书、平台授权「已通过/待开通」类闭环。

## 对队列/摘要的期望（用于人工验收）

- `action_now` / 紧急项里，应**相对弱化**「资源审批待我批复」类建议（除非邮件明确 @ 我）。  
- 对**工时、HR、授权闭环、集团通知**类线程，优先级与解释应**相对上浮**（在有多条候选时）。
